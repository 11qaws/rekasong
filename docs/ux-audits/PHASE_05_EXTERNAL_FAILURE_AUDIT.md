# UX 감사 Phase 05 — 외부 실패·손상 입력 4×30

기준: 외부 장애를 추측하거나 숨기지 않고, 현재 곡·대기열을 의도치 않게 바꾸지 않으며, 스트리머가 기다림·재시도·우회·스킵 중 다음 행동을 고르게 한다.

판정: 반영(현재 처리), 안내(안전 알림), 제약(현 구조로 검증 불가), 후속(별도 설계).

## A. 스트림·OBS 위젯·동기화 30문항

1. 컨트롤러 인터넷 단절 — 안내: 위젯 표시 성공으로 단정하지 않음.
2. OBS 컴퓨터 오프라인 — 제약: heartbeat 전 감지 불가.
3. ntfy relay 접속 실패 — 안내: 로컬 채널·저장소 경로는 계속 사용.
4. 개발 sync 서버 종료 — 안내: POST 실패가 방송 제어를 막지 않음.
5. 위젯이 먼저 열림 — 반영: 마지막 저장 payload를 읽음.
6. 컨트롤러가 먼저 열림 — 반영: 상태 변경마다 payload 갱신.
7. room 누락 — 반영: API가 400 거절.
8. room 불일치 — 반영: 다른 채널 payload 미수신.
9. 서명 키 미생성 — 안내: 로컬 경로로만 동작.
10. 잘못된 공개 키 — 반영: 검증 실패 payload 무시.
11. 변조 relay message — 반영: ECDSA 검증 실패 무시.
12. 오래된 relay message — 반영: timestamp 역행 무시.
13. 중복 message — 반영: 최신 timestamp만 적용.
14. localStorage 불가 — 안내: 예외 후 다른 경로 시도.
15. BroadcastChannel 미지원 — 안내: 저장소·relay 경로 사용.
16. private browsing 저장소 정리 — 제약: 새 room으로 재시작 가능.
17. 위젯 새로고침 — 반영: cache·relay 재수신 시도.
18. 컨트롤러 새로고침 — 반영: 정상 JSON 상태만 복원.
19. 컨트롤러 두 탭 동시 조작 — 후속: 제어권 lease 정책 필요.
20. 위젯 두 개 동시 표시 — 반영: 동일 room 최신 상태 표시.
21. relay rate limit — 제약: 전송 결과 재시도·확인 없음.
22. relay payload 과대 — 후속: 상태 크기 제한 필요.
23. GitHub Pages 함수 API 부재 — 안내: 서버 기능은 endpoint 필요.
24. OBS CORS·네트워크 차단 — 제약: 수신 heartbeat 전 감지 불가.
25. OBS가 오래된 URL 사용 — 제약: 복사 성공은 반영 성공이 아님.
26. room 추측 — 반영: 서명 없는 payload 미적용.
27. 공개 키 URL 누락 — 반영: relay payload 미적용.
28. 시스템 시계 이상 — 후속: 단조 revision 필요.
29. sync API 손상 JSON — 반영: 400 거절.
30. 연결됨 상태 표시 요구 — 제약: heartbeat 전 표시 금지.

13. 영상 정상 종료 — 반영: 자동 다음 곡 정책 적용.
14. 종료 이벤트 지연 — 반영: 곡 ID 동일 시만 처리.

19. 자동 다음 곡 꺼짐 정상 종료 — 반영: 대기열 보존.
20. 자동 다음 곡 켜짐 정상 종료 — 반영: 다음 곡 연결.
21. autoplay 정책 차단 — 후속: 명시적 재생 재시도 제어 필요.
22. iframe 로드 실패 — 안내: 오류 이벤트 안전 경로 사용.
23. 썸네일 로드 실패 — 반영: 위젯 대체 이미지.
24. 제목·채널 누락 응답 — 후속: 결과 스키마 fallback 필요.

26. 검색 API 네트워크 실패 — 반영: URL 우회 안내.
27. 검색 API 비배열 JSON — 반영: 오류 처리로 화면 붕괴 방지.

## C. 로컬 음원·파일 데이터 30문항

1. 0 byte 파일 — 반영: audio 오류로 해당 곡만 스킵.

4. 미지원 코덱 — 반영: source-not-supported 안내.
5. 파일 읽기 중단 — 반영: aborted 안내.
6. 파일 읽기 실패 — 반영: network 오류 안내.
7. 재생 중 waiting — 반영: 한 번만 지연 안내.
8. 50MB 초과 — 반영: 업로드 전 차단.
9. 비오디오 MIME — 반영: 업로드 전 차단.
10. 확장자만 MP3 — 반영: 재생 오류 안전 복구.
11. ID3 태그 없음 — 반영: 수동 입력 유지.

19. 매우 긴 파일명 — 후속: 표시 말줄임 정책 필요.

25. 로컬 음원 정상 종료 — 반영: 자동 다음 곡 동일 적용.

5. song id 없음 — 반영: 저장 상태 제외.

## 이번 회차 우선 반영

- 늦은 YouTube/local audio 종료·오류가 새 곡을 건너뛰지 않도록 현재 곡 ID를 확인한다.
- YouTube 오류 코드와 local audio 오류를 이해 가능한 문장으로 안내하고, 오류 곡만 넘긴다.
- 검색 API는 12초 timeout과 응답 배열 검증을 적용해 URL 직접 입력 우회를 제시한다.
- localStorage 복원 상태는 정규화하며, 새로고침 뒤 재생 불가한 local Blob 곡은 제거한다.
