# Phase 02 — 방송 제어·복구 UX 감사 (30문항)

## 범위와 성공 시나리오

방송 중인 스트리머가 현재 곡·대기열·OBS 위젯을 한 화면에서 안전하게 제어한다. 실수는 되돌릴 수 있거나, 파괴적 동작 전에 의미가 명확해야 한다. 자주 쓰지 않는 설정은 기본 동선을 가리지 않는다.

판정 표기: `통과` / `보완` / `위험`

| # | 감사 질문 | 현황 판정 | Phase 02 결정 |
|---:|---|---|---|
| 1 | 현재 재생 중인 곡이 한눈에 보이는가? | 통과 | ON AIR·현재 곡 카드를 유지한다. |
| 2 | 일시정지와 재생을 즉시 전환할 수 있는가? | 통과 | 헤더·현재 곡의 재생 제어를 유지한다. |
| 3 | 음소거와 볼륨 복원이 빠른가? | 통과 | 직전 볼륨 복원 토글을 유지한다. |
| 4 | 긴 전주를 건너뛸 수 있는가? | 통과 | 탐색 바를 유지한다. |
| 5 | 잘못된 곡을 급히 멈출 때 우발 실행을 막는가? | 통과 | 2단계 확인형 긴급 정지를 유지한다. |
| 6 | 긴급 정지가 무엇을 지우는지 분명한가? | 보완 | 다음 회차에서 버튼 보조 설명을 검토한다. |
| 7 | 대기열 순서를 마우스로 바꿀 수 있는가? | 통과 | 드래그 대상선 피드백을 유지한다. |
| 8 | 대기열에서 곡을 삭제할 때 실수를 되돌릴 수 있는가? | 보완 | Undo 방식은 상태·위젯 동기화와 함께 다음 회차에서 설계한다. |
| 9 | 이전 곡을 다시 맨 앞에 예약할 수 있는가? | 통과 | 재예약 기능을 유지한다. |
| 10 | 위젯에 남은 이력을 지울 때 확인하는가? | 통과 | 삭제 확인 대화상자를 유지한다. |
| 11 | 자동 다음 곡이 기본 흐름에서 불필요한 판단을 강요하는가? | 위험 | 접힌 `재생 옵션`으로 이동하고 현재 상태를 요약한다. |
| 12 | 자동 다음 곡의 의미와 켜짐/꺼짐 상태가 보이는가? | 보완 | 옵션 내부에 행동 결과를 설명한다. |
| 13 | 자동 다음 곡 상태가 실제 재생 로직과 같은가? | 통과 | 동기화 상태 하나만 사용한다. |
| 14 | 위젯 URL 설정이 매 방송마다 필요한 일처럼 보이는가? | 보완 | `처음 한 번만` 문구를 접힌 요약에 넣는다. |
| 15 | 위젯 주소 복사 성공·실패를 사용자가 알 수 있는가? | 위험 | 브라우저 alert 대신 앱 토스트와 복사 실패 안내를 사용한다. |
| 16 | 위젯 미리보기가 핵심 제어를 밀어내는가? | 통과 | 접힌 위젯 설정 내부에만 둔다. |
| 17 | 현재 재생 곡이 없을 때 다음 행동이 안내되는가? | 보완 | 1→2단계 안내 문구를 추가한다. |
| 18 | 대기열이 비었을 때 자동 다음 곡이 어떻게 되는지 오해하지 않는가? | 통과 | 빈 상태와 옵션 설명을 함께 표시한다. |
| 19 | 단축키가 입력창 편집을 방해하지 않는가? | 통과 | 입력 요소에서는 전역 단축키를 무시한다. |
| 20 | 버튼의 파괴성(삭제·정지)이 색·문구로 구분되는가? | 통과 | danger 스타일과 확인 단계를 유지한다. |
| 21 | 토스트가 곡 선택·복사·취소의 결과를 알려 주는가? | 보완 | 위젯 복사도 토스트 체계에 편입한다. |
| 22 | 위젯이 늦게 열려도 현재 상태를 복원하는가? | 통과 | 방별 상태 복원을 유지한다. |
| 23 | 로컬 개발과 정적 배포의 동기화 차이를 숨기지 않는가? | 보완 | 개발 로그에 기록했고 연결 상태 표시는 다음 회차 대상이다. |
| 24 | 목업 노래책이 실제 신청곡처럼 취급되지 않는가? | 통과 | 데모 배너를 유지한다. |
| 25 | 긴 제목·태그가 대기열 조작 버튼을 밀어내지 않는가? | 통과 | 말줄임과 고정 버튼 영역을 유지한다. |
| 26 | 작은 화면에서 제어가 지나치게 촘촘해지지 않는가? | 보완 | 실제 모바일 viewport 검증은 다음 회차 대상이다. |
| 27 | 새로고침 뒤 자동 다음 곡·볼륨이 예측 가능하게 남는가? | 통과 | 동기화 상태·로컬 저장을 유지한다. |
| 28 | 복사 권한이 막힌 환경에서 대안을 안내하는가? | 보완 | 실패 토스트에 수동 복사 안내를 제공한다. |
| 29 | 접근성 도구가 위젯 설정·재생 옵션의 접힘 상태를 읽을 수 있는가? | 통과 | 네이티브 `details/summary`를 사용한다. |
| 30 | 방송 제어 화면에 핵심과 고급 기능의 위계가 있는가? | 보완 | 재생 옵션·위젯 설정을 접고 핵심 제어를 먼저 둔다. |

## Phase 02 구현 우선순위

1. **P0 — 복사 피드백:** 복사 성공과 실패를 앱 토스트로 안내한다.
2. **P1 — 판단 부담 축소:** 자동 다음 곡을 접힌 재생 옵션으로 옮기고 상태·효과를 설명한다.
3. **P1 — 빈 상태 안내:** 재생 전에는 1→2단계 행동을 짧게 안내한다.
