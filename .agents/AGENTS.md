# Global Workspace Rules

## 1. UI/UX 및 디자인 철학 (모든 UI 프로젝트 공통)
- **사용자 편의성 최우선:** 모든 검토는 UI, UX 측면에서 사용자의 편의성을 최우선으로 고려한다.
- **초심자 중심 설계 (Beginner-first):** 항상 가장 초심자의 입장에서 디자인을 구성해야 한다.
- **시각적 흐름 (Visual Flow):** UI는 시각적으로 단방향(위에서 아래로, 왼쪽에서 오른쪽으로)으로 구성되어야 한다.
- **웹의 관습 준수:** 모든 UI는 웹의 보편적 관습을 최대한 따른다.
- **대안 제시:** 항상 장점과 단점을 분석하고 대안을 제시한다.

## 2. 시나리오 기반 심층 검토 (Scenario-based Review)
- **Before/After 검토:** 설계 변경 시, 사용자의 시나리오와 행동 패턴을 논리적으로 가정하여 입체적으로 비교 검토한다.
- **리스크 헷징(Risk Hedging):** 발생할 수 있는 '논리적 오류', '시각적 착각', '의도하지 않은 프로세스'를 예외 없이 확인한다.

## 3. Rettostock 구조 준수 (프로젝트 특화)
- 본 프로젝트(rekasong)의 모든 폴더 구조, 컴포넌트 분리, 네이밍 규칙, UI/UX 철학은 인접 프로젝트인 **`rettostock_repo`**를 1순위 기준으로 삼는다.
- 거대 컴포넌트를 지양하고, `pages`, `components`, `hooks`, `utils` 등 역할별로 분리한다.
- `Glassmorphism` 등 고급스러운 테마(Themes)를 기반으로 디자인한다.
